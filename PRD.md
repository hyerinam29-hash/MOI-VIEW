## 1. 개요 및 긴급 목적 (Executive Summary)

**'MOI VIEW (모이뷰)'**는 여행이 어려운 이들에게 대리 만족을 선사하기 위한 서비스입니다.
본 PRD는 기존 7주 분량의 MVP 계획을 **단 하루(1일)**로 단축하여, 핵심 가치인 **"웹에서 내 사진을 여행지 배경과 즉시 합성하여 보여주는 것이 기술적으로 가능한가?"**를 검증하는 **PoC(Proof of Concept, 개념 증명)** 수준의 초단기 구현을 목표로 합니다.

**긴급 제약 사항 (Crucial Constraint):**
*   **개발 기간:** 24시간 이내 (1 Day Build)
*   **범위 축소:** 사용자 인증, DB, 백엔드 서버 구축, **동영상 처리 기능 전면 제외**. 오직 프론트엔드와 외부 이미지 API 연동에만 집중한다.

## 2. 타겟 오디언스 (Target Audience)

*   (PoC 단계에서는 특정 타겟보다는 기능 구현 자체에 집중하나, 궁극적인 타겟은 아래와 같음)
*   건강상/경제적 이유로 여행이 불가능하여 정서적 환기가 필요한 사용자.

## 3. 핵심 문제와 1일 해결책 (The 1-Day Solution)

*   **문제:** 여행지에 가고 싶지만 갈 수 없다. 내 모습이 그곳에 있는 사진이라도 갖고 싶다.
*   **1일 해결책:** 복잡한 절차 없이 웹페이지에 접속하여, 내 사진 파일을 올리면 즉시 배경이 제거되어 여행지 사진 위에 합성된 결과를 눈으로 확인하고 다운로드한다.

## 4. 프로젝트 목표 및 원칙 (Goals & Principles)

**1일 PoC 목표:**
*   군더더기 없는 단일 페이지 웹 애플리케이션(SPA) 배포.
*   사용자가 사진 업로드 시 외부 AI API가 정상 동작하여 배경이 제거되는지 검증.
*   제거된 인물 이미지와 준비된 배경 이미지가 웹상에서 합쳐져 보이는지 검증.

**원칙:**
*   **작동 우선 (Make it Work):** 디자인 완성도나 예외 처리보다 "일단 돌아가는 것"이 최우선이다.
*   **NO Backend:** 별도의 백엔드 서버 구축 없이 프론트엔드에서 해결하거나 최소한의 Serverless Function(Next.js API Routes 등)만 사용한다.
*   **동영상 포기:** 1일 내 구현 불가능한 동영상 기능은 과감히 포기하고 **사진**에만 집중한다.

## 7. 사용자 흐름 (1-Day User Flow)

아주 단순한 선형 구조입니다.

1.  **접속:** 웹페이지 접속
2.  **선택:** (옵션) 화면에 보이는 배경 3개 중 맘에 드는 것 클릭 (기본값 1번 지정)
3.  **업로드:** "내 사진 선택" 버튼 클릭 -> 파일 열기
4.  **대기 & 결과:** (로딩 인디케이터 표시) -> API 처리 후, 화면 중앙에 배경과 내 모습이 합성된 이미지가 나타남.
5.  **저장:** 하단의 "이미지 다운로드" 버튼 클릭하여 소장.

## 8. 권장 기술 스택 (1-Day Stack)

빠른 개발 및 배포를 위해 설정이 최소화된 스택을 권장합니다.

*   **Frontend & Deployment:** Next.js (Vercel 배포) - *API Routes를 사용하여 외부 API 키를 숨기는 용도로 활용 가능.*
*   **AI API:** remove.bg API 또는 Clipdrop API (가장 연동이 쉽고 문서화가 잘 된 곳 선택)
*   **Synthetic Lib:** HTML Canvas API 직접 사용 또는 html2canvas (다운로드 구현용)
